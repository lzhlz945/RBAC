权限控制：包括了2种 1、操作权限   2、数据权限

操作权限：不同的用户登录成功以后能够看到的页面中的功能数量和可以点击的功能数量是不一样
          例如销售员登录后可以添加和查看订单，而销售经理登录以后可添加和查看订单外加
	  可以修改订单和删除订单

数据权限：不同的用户登录成功以后同一个功能看到的数据内容是不一样的，例如销售员能看到
          自己的订单信息，而销售经理可以看到自己的订单以及手下所有员工的订单信息

RBAC（基于角色的访问控制）：RBAC就是专门用于实现操作权限控制的一种设计模式（设计方式）

RBAC一共分为4个级别
1、 RBAC0 基础的权限控制
2、 RBAC1 在RBAC0的基础上添加了角色的继承功能，例如销售经理就应该继承销售员那么销售经理
          就会拥有销售员的所有功能外加销售的经理的特殊功能
3、 RBAC2 在RBAC0的基础上添加了互斥的功能，例如一个员工拥有了会计的角色，那么它就不能再拥有
          出纳的角色
4、 RBAC3 即拥有RBAC1也同时拥有RBAC2的功能


RBAC的权限管理包括的功能：导航菜单的控制，页面按钮元素的显示控制，权限的拦截，以及权限的分配

至少需要5账号数据库表
1、permission （权限表/许可表）
作用：用于记录当前应用中所有需要被添加拦截控制的功能，可以提供独立的增删改查来
      对表中数据进行维护，但是必须至少要将一些核心的功能在项目部署时就初始化到
      数据库中，例如权限的添加以及权限的分配以及用户的添加等

表结构
id    name            url               code  [ type   parentId   icon  ...]
1    添加订单   /private/addOrder       tjdd
2    查询订单   /private/getOrder       cxdd
3    修改订单   /private/updateOrder    xgdd
4    删除订单   /private/delOrder       scdd

字段说明
id   主键  自动递增
name 权限名 用于在页面中让用户观看的
url  这个权限的请求路径 与code二选一即可获全部拥有 用于实现权限的拦截以及动态生成请求地址
code 这个权限的编码与url二选一即可或全部拥有 用于实现权限的拦截



2、role  角色表
作用：用于记录当前应用中所有的可以分配个用于的具体的角色信息，可以提供独立的增删改查
      来维护表中的额数据，但是必须至少要将一些核心的角色在项目部署时初始化到数据数据库中
      例如 管理员角色

表结构
id       name
1      普通员工
2      销售员
3      销售经理
4      出纳
5      会计

字段说明
id    主键自动递增

name 角色名称用于在页面中让用户观看的

注意：如果需要实现RBAC1或RBAC2的功能需要额外在这个表中添加字段或使用多对多表进行关联
     如果需要实现单一继承或单一排斥则只需要添加继承的字段和排斥字段即可然后显示自关联设计
     如果需要实现多继承或多排斥则需要创建多对多关联表中需要实现role表示继承和排斥关系


3、 user   用户表
作用：用于记录当前系统中可以操作系统功能的具体用户，需要利用这些数据进行登录然后才能操作
      系统中的功能 可以提供独立的增删改查来维护表中的数据，但是必须在项目部署时初始化一个
      必要的数据例如 管理员的账号和密码

表结构
id     username     password
1      zhangsan     123456
2      lisi         123456      
3      wangwu       123456
4      zhaoliu      123456
5      chenqi       123456
6      zhouba       123456

字段说明

id        主键自动递增
username  用户账号 用于登录的  账号应该具有唯一并添加索引
password  用户密码 用于登录的



4、 role_permission  角色和权限的多对多关联表
作用：用于记录系统某个角色所具有的具体功能

表结构
id    role_id     permission_id
1      2               1
2      2               2
3      3               1
4      3               2
5      3               3
6      3               4

字段说明
id              主键自动递增  可以字段 如果没有这个列建议将角色id和权限id设置为联合主键
role_id         角色id 外键 用于存放角色表中的主键
permission_id   权限id 外键 用于存放权限表中的主键


5、 user_role  用户和角色的多对多关联表
作用：用于记录系统中某个用户所以拥有的角色

表结构
id     user_id     role_id
1        1           1
2        2           2
3        3           2
4        4           3 
5        5           4
6        6           5
id              主键自动递增  可以字段 如果没有这个列建议将角色id和用户id设置为联合主键
user_id         用户id 外键 用于存放用户表中的主键
role_id         角色id 外键 用于存放角色表中的主键


注意：这2张多对多的关联表不需要提供独立的增删改查，来维护表中的数据，在项目部署时必须
      要将一些必须的数据初始化到数据库中，例如管理员用户与管理员角色的管理，以及管理员
      角色和一些基础功能的关联
